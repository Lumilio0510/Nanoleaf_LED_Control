# AI 场景感知灯光系统 — 脑暴会议总结

> 日期：2026-05-25
> 背景：基于已有的 LED Control（Nanoleaf 桌面控制）项目，探讨商业化方向和技术方案

---

## 一、项目背景回顾

现有项目是一个 Electron 桌面应用，可通过 AI Agent 用自然语言控制 Nanoleaf 灯板、生成灯效并保存为可复用的 Skill。项目技术架构成熟（Electron + React + MUI + 多轮 Tool Calling Agent），已有完整的设备管理、AI 对话、Skill 系统。

---

## 二、产品经理视角的项目评价

### 优势
- **架构成熟：** Tool Calling + Skill 系统的 AI 灯效生成能力是核心竞争力
- **体验完整：** 自动发现设备、流式对话、快速命令等细节到位
- **AI 能力强：** 自然语言转复杂灯效参数，这是官方 App 没有的能力

### 待解决的问题
- **硬件绑定：** 锁定 Nanoleaf，市场规模受限
- **场景错位：** 灯控的天然场景在手机上，而产品是桌面端
- **推广难题：** 个人开发者难以做大规模消费者推广

---

## 三、商业化方向决策

### 排除的方案
- **心理咨询室（场景 A）：** 场景过于私密，推广难度大 → 放弃
- **C 端直接推广：** 个人开发者做消费市场不现实 → 放弃

### 选定方向

**B2B 企业方案 → 最终授权给 Nanoleaf**

先做 B2B 积累客户和案例，再拿数据和产品去与 Nanoleaf 谈技术授权。

### 两个 B2B 细分市场

#### 市场 1：高端商务谈判/接待室
- 酒店行政酒廊、律所会客室、企业 VIP 接待厅、私人会所
- 核心价值：灯光在谈判不同阶段"替人说话"——破冰用暖色、谈判用中性光、签约用庆祝灯效
- 付费意愿高：这些场所装修预算 10 万+，体验升级敏感度高

#### 市场 2：混合办公共享空间
- 联合办公空间、企业开放区、酒店多功能区、展厅
- 核心价值：同一空间在不同时段自动适配不同用途（工作→社交→休息→清洁）
- 运营方用灯效做差异化竞争

### 产品交付形式

**方案 A：纯软件订阅（SaaS）** → 选定

- 客户自备 Nanoleaf 面板 + 常开电脑
- 部署我们的桌面客户端
- 定价 $49-99/月/空间
- 优势：交付最轻，无需额外硬件，可远程部署

### 商业化时间线

| 阶段 | 时间 | 目标 |
|------|------|------|
| 短期 | 1-3 个月 | 完成 MVP，3-5 家企业试点 |
| 中期 | 3-6 个月 | 积累客户案例，验证付费意愿 |
| 长期 | 6-12 个月 | 以产品+案例与 Nanoleaf 洽谈授权 |

## 四、实现方案 — 场景感知灯光系统

### 核心概念

通过本地音频分类 + 窗口检测 + 云端 LLM 仲裁，让灯光自动感知当前场景并切换对应灯效。用户不需要打开客户端界面来输入需求。

### 架构总览

```
[会议室 PC]
  │
  ├─ 音频采集模块 ── 本地轻量模型分类（安静/讲话/多人/音乐）
  ├─ 窗口检测模块 ── Windows API 识别前台应用
  ├─ 天气集成 ──── 和风天气 API
  │
  └─→ 场景推理引擎
        ├─ 第一层：本地规则（高置信度直接输出）
        │    例：腾讯会议 + 多人对话 → 会议中（确信）
        │
        └─ 第二层：云端 LLM（低置信度时辅助判断）
             上下文中不含原始录音/截图
             仅含分类标签 + 时间 + 天气 + 窗口类别
              → 最终目标：灯效映射
```

### 技术原理分层

#### 1. 感知层
- **音频：** 每 15-30 秒采集 3 秒片段，本地提取特征后分类，不上传原始录音
- **窗口：** 轮询 Windows 前台进程（腾讯会议、钉钉、飞书、Zoom、Chrome 等）
- **天气：** 国内免费 API（和风天气/彩云天气）

#### 2. 推理层（两层架构）
- **本地规则：** 处理明确场景（如"腾讯会议 + 多人对话声"= 会议中），零延迟
- **云端仲裁：** 本地不确定时，发上下文标签（不含原始数据）给 LLM API 做综合推断

#### 3. 执行层
- 场景 → 映射到已有 Skill 系统 → 调用 `skill:execute` → Nanoleaf 灯板

### 场景灯效映射（预置方案）

| 场景 | 灯效 |
|------|------|
| 会议中 | 中性色温 4000K + 明亮 |
| 演示/汇报 | 暖白 + 柔和 |
| 专注工作 | 冷白 5000K + 高亮度 |
| 社交/接待 | 暖色渐变动态 |
| 休息/放松 | 低亮度暖色 |
| 离开 | 待机/关闭 |

### MVP 范围

**v1（MVP）：**
- 本地音频分类（安静/单人讲话/多人对话/音乐）
- 窗口检测（常见国产应用进程）
- 本地规则引擎 + 云端 LLM 推理
- 预置 6 种场景灯效映射
- 天气集成
- 基础设置面板（开关感知、选择麦克风、配置 LLM API Key、隐私选项）
- 单空间部署

**v2（后续）：**
- 飞书/钉钉开放 API 对接
- 多空间管理
- 管理后台 / 使用数据看板
- 场景识别准确率优化

**v3（远期）：**
- 与 Nanoleaf 授权合作

### 隐私设计
- 不上传原始音频，仅传分类标签
- 不上传窗口标题，仅传进程类别
- 所有采集模块有独立开关
- 系统托盘显示感知状态指示灯

---

## 五、关键结论

1. **不做 C 端**，走 B2B → Nanoleaf 授权的路径
2. **不做心理咨询场景**，聚焦商务谈判/接待室 + 共享办公空间
3. **纯软件 SaaS** 交付，客户自备硬件
4. **本地 + 云端两层架构**，兼顾隐私和推理能力
5. **先做 MVP 验证**，积累案例后再和 Nanoleaf 谈判
