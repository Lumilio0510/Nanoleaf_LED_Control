# Canvas AI 生成质量优化设计

## 概述

优化 Canvas AI 灯板生成功能，目标是让 LLM 生成的图形更贴近用户描述，同时减少生成等待时间。保持当前坐标生成方式不变，不引入布局引擎。

## 问题分析

### 质量问题的根源

1. **参考方案被复制而非参考**：`loadReferenceDesigns()` 向 LLM 展示最多 3 个参考方案的完整 JSON 坐标，并指示"参考坐标排列规律"。LLM 倾向于直接复制参考方案的布局拓扑，忽略用户当前描述的主题。

2. **形状指导不够精确**：当前 shape guide 只有笼统的文字描述（"上方用两个斜放的三角形形成内凹弧"），缺少面板数量范围、比例关系、构造步骤等可操作的指导。

3. **缺少预先规划**：LLM 直接输出坐标，没有先构思布局的机会。

### 速度问题的根源

修正循环最多 5 次，每次都需要完整调用 LLM，等待时间过长。

## 优化方案

### 1. 参考方案改为知识萃取

**改动位置**：`canvas-agent.service.ts` — `loadReferenceDesigns()`

不再将参考方案的完整 JSON 坐标放入提示词。改为预先分析所有已有设计，提取抽象特征，以知识库形式呈现。示例输出：

```
## 设计知识库

从已有 15 个设计方案中提炼：

翅膀/翼类（3 个方案）：
  - 面板数 12-21 块，左右完全对称
  - 中心用六边形做连接枢纽，向两侧水平展开
  - 整体横向 500-700px，纵向 300-400px
  - 常用 6-8 对三角形 + 2-4 个六边形 + 小三角形填充

爱心类（2 个方案）：
  - 面板数 10-15 块，左右对称
  - 底部收尖，中部饱满，顶部 V 形内凹
  - 整体约 250×300px，以三角形为主体

花朵类（3 个方案）：
  - 面板数 5-8 块，中心辐射对称
  - 中心一个六边形，周围 5-6 个三角形放射排列
  - 整体约 200×200px，紧凑型
```

**关键变化**：
- 展示的是跨方案统计规律（可迁移的知识），而非单个方案的精确坐标（可复制的模板）
- 提示词指示从"参考坐标排列规律"改为"了解不同主题的常用布局方式"

**函数变更**：
- `loadReferenceDesigns()` 改为 `extractDesignKnowledge()`，按主题分类聚合统计
- 使用 `design.name` 中的关键词进行主题分类（翅膀、爱心、花朵等）
- 对每个主题统计面板数范围、常用面板类型组合、对称性、尺寸范围

### 2. 增强形状构造指南

**改动位置**：`canvas-agent.service.ts` — `SYSTEM_PROMPT_BASE` 中的形状指南部分

为每个形状增加：
- 推荐面板数量范围
- 构造步骤（从核心到外围）
- 比例关系（宽高比）
- 关键特征的具体实现方式

示例（爱心）：

```
### 爱心 (10-15 块面板)
步骤：
1. 底部：1 个 triangle rot=180（尖角朝下）在最底端
2. 中段左右弧：各 2-3 个 panel（hexagon 或 triangle）形成弧形向外展开
3. 顶部 V 口：两个 triangle 相对放置形成内凹
4. 填充：用小三角形填补弧形之间的空隙
要点：底部尖、中部饱满、顶部有 V 形内凹，宽度 ≈ 高度
```

保留原有的 **配色指南**，按主题推荐色系。

### 3. 输出前先描述布局计划

**改动位置**：`canvas-agent.service.ts` — `SYSTEM_PROMPT_BASE` 中的输出格式部分

在提示词中要求 LLM 在输出 JSON 数组之前，先写一句话描述构图思路。例如：

```
我将设计一个苹果造型：中心用六边形构建饱满的主体，顶部微凹底部微收。
```

JSON 提取逻辑 `extractJson()` 已使用 `/\[[\s\S]*\]/` 正则提取数组，会自动跳过前面的文字，无需修改解析代码。

### 4. 减少修正循环

**改动位置**：`canvas-agent.service.ts` — `MAX_ATTEMPTS` 和修正逻辑

- `MAX_ATTEMPTS` 从 5 改为 3
- 第 3 次无论是否有问题都直接返回（不再强制修正）
- 这样最坏情况等待时间减少约 40%

### 5. 改进修正反馈

**改动位置**：`canvas-agent.service.ts` — `buildErrorFeedback()` 和 `buildConnectivityFeedback()`

修正反馈信息增加更具体的修改建议：

- 重叠：不仅指出哪些面板重叠，还建议移动方向（"建议将 #X 面板向上移动约 77px"）
- 不连通：不仅指出分组，还建议靠近哪个面板的哪条边

## 改动范围

仅涉及一个文件：`src/main/canvas-agent.service.ts`

无新增依赖，无架构变化，无 UI 变化。

## 实施步骤

1. 修改 `loadReferenceDesigns()` → `extractDesignKnowledge()` — 知识萃取方式
2. 增强 `SYSTEM_PROMPT_BASE` 中的形状构造指南
3. 在输出格式中增加"先描述计划"的要求
4. `MAX_ATTEMPTS` 从 5 改为 3
5. 改进修正反馈信息
