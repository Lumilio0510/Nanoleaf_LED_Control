# Softwaves LED Control 设计文档

**日期**: 2026-05-13
**版本**: V1

---

## 概述

基于 Electron 的桌面客户端工具，通过局域网 WiFi 连接并控制 LED 灯板，集成 AI Agent 能力，根据开放控制 API 文档自动生成可复用的灯效方案（Skill）。

- **使用者**: 个人单用户
- **平台**: Windows（Electron）
- **连接方式**: WiFi 局域网 + RESTful HTTP API

## 模块一：LED 连接管理

### 设备发现
- 手动输入 IP:端口
- 局域网扫描（UDP 广播 / IP 段扫描）
- 发现后可一键保存

### 连接状态
- 状态机：未连接 → 连接中 → 已连接 → 异常
- 心跳保活：每 10 秒 ping，连续 3 次失败视为断线
- 断线自动重连（可配置次数和间隔）

### 设备配置存储
- 存为 JSON 文件，主进程管理
- 支持多设备：设备名、IP、端口、备注

### IPC 接口

| 接口 | 说明 |
|---|---|
| `scanNetwork()` | 扫描局域网发现设备 |
| `connect(deviceId)` | 连接设备 |
| `disconnect()` | 断开连接 |
| `getDevices()` | 已保存设备列表 |
| `addDevice(config)` | 新增设备 |
| `removeDevice(deviceId)` | 删除设备 |
| `getDeviceStatus()` | 当前连接状态 |

## 模块二：灯效控制

### 功能
- 开关灯
- 亮度调节（0-100%）
- 颜色设置（RGB / 取色器）
- 灯效列表切换（从 API 文档解析，不做硬编码）
- 灯效参数调节（速度、颜色、方向等，由灯效定义决定）

### IPC 接口

| 接口 | 说明 |
|---|---|
| `switch(on/off)` | 开关灯 |
| `setBrightness(0-100)` | 调节亮度 |
| `setColor(r,g,b)` | 设置颜色 |
| `applyEffect(effectId, params)` | 应用灯效 |
| `getEffectList()` | 可用灯效列表 |

## 模块三：Skill 系统

### Skill 定义

Skill = 一个 JSON 文件，包含三部分：

```json
{
  "meta": {
    "id": "skill_001",
    "name": "红蓝渐变呼吸灯",
    "description": "在红色和蓝色之间缓慢呼吸渐变",
    "tags": ["呼吸", "渐变"],
    "version": 1,
    "createdAt": "2026-05-13T10:00:00Z"
  },
  "params": [
    {
      "key": "speed",
      "label": "速度",
      "type": "range",
      "min": 1,
      "max": 10,
      "default": 3
    },
    {
      "key": "colorA",
      "label": "颜色A",
      "type": "color",
      "default": "#FF0000"
    },
    {
      "key": "colorB",
      "label": "颜色B",
      "type": "color",
      "default": "#0000FF"
    }
  ],
  "mapping": {
    "endpoint": "POST /effect/breathe",
    "bodyTemplate": {
      "mode": "breathe",
      "speed": "{{params.speed}}",
      "colors": ["{{params.colorA}}", "{{params.colorB}}"],
      "transition": "smooth"
    }
  }
}
```

- **meta**: 描述信息，供 UI 展示和搜索
- **params**: 用户可调参数，决定 UI 渲染什么控件（滑块、取色器、选择器等）
- **mapping**: 参数到 HTTP API 请求的映射规则，`{{params.key}}` 模板替换

### 执行流程

```
用户输入"帮我做个红蓝呼吸灯"
        │
        ▼
    Agent（主进程）
  拼接 System Prompt（API 文档 + Skill Schema）
        │
        ▼
    LLM 返回 Skill JSON
        │
        ▼
    主进程解析 → 存入 skills/ 目录
        │
        ▼
    渲染进程展示 Skill 卡片（预览 / 编辑 / 执行）
        │
        ▼ 用户点击执行
    Mapping 引擎替换 {{params.xxx}} → HTTP 请求到灯板
```

### Skill 库管理

- 存储位置：`skills/` 目录，每个 Skill 一个 JSON 文件
- 支持操作：新建 / 编辑 / 删除 / 复制 / 导出
- 渲染进程：卡片网格展示，支持搜索、标签过滤

### 主进程 Agent 编排

```
用户消息 → 意图判断
              ├─ 闲聊 → 直接回复
              ├─ 生成灯效 → System Prompt(API文档+Skill Schema) → LLM → Skill JSON
              └─ 直接控制 → System Prompt(API文档+当前状态) → LLM → API调用指令
```

### 多模型支持

- LLM 适配器接口：`chat()` / `streamChat()`
- V1 实现：OpenAI 适配器（兼容 API）、Ollama 适配器
- API Key 存储在主进程本地，不暴露给渲染进程
- 设置页面配置模型类型、端点、密钥

### 聊天历史持久化

- 按会话存储为 JSON 文件
- 支持：新建会话、查看历史会话、切换会话
- Agent 可引用当前会话上下文

### 快捷指令

- 用户可自定义快捷指令模板（如 `"帮我做一个 {{theme}} 灯效"`）
- 内置默认指令：随机灯效、关灯、调至最亮等
- 点击后自动填入聊天输入框或直接发送

## V1 功能范围

### 包含
- 设备连接管理（手动 + 局域网扫描）
- 基础控制（开关、亮度、颜色）
- 内置灯效切换、参数调节

- Skill 生成、保存、执行
- Skill 库浏览和管理
- 聊天历史持久化
- 断线自动重连

### 暂不包含
- 场景模式
- 更多 LLM 适配器（Claude 等）
- 自动更新
