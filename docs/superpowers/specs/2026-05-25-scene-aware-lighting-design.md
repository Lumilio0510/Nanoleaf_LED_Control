# AI 场景感知灯光系统设计文档

## 1. 产品定位

面向国内 B2B 市场的 AI 场景感知灯光系统。通过在会议室、商务接待室、共享办公空间等商业场所的电脑上部署轻量客户端，利用本地音频识别 + 窗口检测 + 云端 LLM 推理，自动感知当前场景并切换 Nanoleaf 灯效。

## 2. 商业目标

- **短期（1-3 个月）：** 完成 MVP，在 3-5 家合作企业试点
- **中期（3-6 个月）：** 积累客户案例，验证付费意愿，打磨产品体验
- **长期（6-12 个月）：** 以产品 + 案例为基础，与 Nanoleaf 官方洽谈技术授权合作

## 3. 目标市场

- **高端商务谈判/接待室：** 酒店行政酒廊、律师事务所会客室、企业 VIP 接待厅、私人会所
- **混合办公共享空间：** 联合办公空间、企业内部开放式茶水间/休息区、酒店大堂多功能区、展厅/画廊

## 5. 模块设计

### 5.1 音频采集模块

**功能：**
- 调用 Windows 麦克风设备，获取系统音频输入
- 每 15-30 秒采集一个 3 秒音频片段
- 在本地提取音频特征：音量均值、频谱分布、语音活动检测
- **不上传原始录音**，只传分类标签

**本地分类模型：**
- 使用轻量级模型（如 YamNet、Speech Commands 级），在 CPU 上实时运行
- 分类结果：安静 / 单人讲话 / 多人对话 / 有背景音乐

### 5.2 窗口检测模块

**功能：**
- 通过 Windows API 轮询（每 5-10 秒）获取当前前台窗口信息
- 检测：进程名、窗口标题、是否为全屏
- 用户可在设置中控制检测粒度（例如：仅检测进程名、不检测标题）

**重点关注应用：**
- 腾讯会议、钉钉、飞书、Zoom、Chrome、Edge、PowerPoint、Word、Excel
- 屏幕保护/锁屏状态（判断用户是否离开）

### 5.3 场景推理引擎

**第一层 — 本地规则引擎：**

| 条件组合 | 推断场景 | 置信度 |
|----------|----------|--------|
| 腾讯会议/钉钉/Zoom + 多人对话 | 会议中 | 高 |
| 腾讯会议/钉钉/Zoom + 安静 | 会议中（静音） | 中 |
| PPT/Chrome + 单人讲话 | 演示/汇报中 | 高 |
| 无语音活动 + 白天 + 文档/IDE | 专注工作 | 中 |
| 无语音活动 + 屏保/锁屏 | 离开 | 高 |
| 夜间 + 安静 + 休闲应用 | 休息/放松 | 中 |
| 多人对话 + 非会议应用 | 社交/接待 | 中 |

置信度高的场景直接输出，不用经过云端。

**第二层 — 云端推理：**

- 触发条件：规则引擎置信度为"中"或无法匹配
- 上下文内容（不含原始数据）：
  - 场景分类标签 + 置信度
  - 窗口类别（"会议应用"、"文档"、"浏览器"等，不含具体标题）
  - 当前时间 + 星期几
  - 天气状况（晴天/阴天/雨天）
  - 最近一次场景切换（避免抖动）
- 返回结果：最可能的场景 + 推荐灯效参数
- LLM 提供商：复用现有 LLM 设置（OpenAI 兼容 API / Ollama）

### 5.4 灯效映射执行器

- 每个场景关联一个 Skill（复用现有 Skill 系统）
- 预置场景灯效方案（可自定义）：
  - 会议中 → 中性色温（4000K）+ 明亮
  - 演示/汇报 → 暖白 + 柔和
  - 专注工作 → 冷白（5000K）+ 高亮度
  - 社交/接待 → 暖色渐变动态
  - 休息/放松 → 低亮度暖色
  - 离开 → 待机/关闭
- 用户可在设置中自由关联场景与 Skill
- 场景切换自动调用 `skill:execute`

### 5.5 天气集成

- 使用国内免费天气 API（和风天气 / 彩云天气）
- 根据 IP 或手动配置城市
- 影响维度：阴雨天增加暖色调权重，晴天保持明亮

## 6. MVP 与后续规划

### v1（MVP）

- 音频场景分类：安静 / 单人讲话 / 多人对话 / 有背景音乐
- 窗口检测：常见国产应用进程识别
- 本地规则引擎 + 云端 LLM 推理
- 预置 6 种场景灯效
- 基础设置面板
- 天气集成
- 单空间部署

### v2（计划）

- 飞书/钉钉开放 API 对接（会议状态事件做强信号）
- 多空间/多会议室管理
- 管理后台 / 使用数据看板
- 场景识别准确率提升（调优本地模型）

### v3（远期）

- 屏幕画面弱采集（仅做活动类型识别，不传画面内容）
- 批量部署方案
- 与 Nanoleaf 授权合作

## 7. 隐私安全设计

- 原始音频不上传，仅传场景分类标签
- 窗口标题默认不上传，仅传进程名/进程类别
- 所有采集模块设有独立开关，用户可以随时关闭
- 云端推理上下文不含可识别身份的信息
- 系统托盘图标提示当前感知状态（指示灯）

## 8. 技术选型说明

| 层面 | 选型 | 理由 |
|------|------|------|
| 客户端框架 | Electron（现有） | 项目已成熟，直接复用 |
| 音频处理 | Web Audio API + 轻量 ONNX 模型 | 在 Electron 的 Chromium 引擎中运行，无需额外原生依赖 |
| 窗口检测 | Node.js `active-win` 或 `win32-api` 原生模块 | 纯本地，低开销 |
| 天气 API | 和风天气（免费版） | 国内可用，免费额度足够 |
| 云端推理 | OpenAI 兼容 API（复用现有 LLM 配置） | 客户已有配置 |
